## 12:47 UTC — v0.3.1 F5: frame scores seed pass (pulled forward)

**Did:** Discovered the actual blocker behind F2: `frame_scores` was empty (0 rows) in the live DB. The plan ordered F2 (map replot) before F5 (frame scores seed), but with no scores, the map plots nothing — so the "selections aren't reaching the query" hypothesis was untestable. Pulled F5 forward. Added `seedFrameScores` to `src/db/seed-data.ts` as a `Record<slug, Record<frameName, 1..5>>`, hand-scoring all 30 seeded companies against all 5 scale-kind frames. Anchored each score against the frame's `lowLabel`/`highLabel` pair (e.g. UK-pigeonhole risk 1 = clearly international, 5 = very UK-bound; faculty-ai → 5; cognition → 1). Wired an upsert block in `src/db/seed.ts` that builds slug→id and name→id maps from the just-upserted rows, then `onConflictDoUpdate` on the composite `(company_id, frame_id)` PK so re-seeding overwrites cleanly. Reseeded Neon (`set -a; . .env.local; npm run db:seed` → 150 frame_scores upserted). `tsc --noEmit` clean. Committed (`2426766`) and pushed.

**Why this shape:** Storing the scores in `seed-data.ts` as a slug-keyed object (rather than nesting them on each `SeedCompany`) keeps companies and their scores edit-able independently — the company list is editorial prose about the company; the score matrix is editorial prose about the *fit*. Looking up frame ids by name at seed-time (rather than freezing a frame_id constant) means the frame seed and the score seed stay decoupled — if frame sort_index or name changes, scores follow as long as the name string matches. Using composite-PK `onConflictDoUpdate` (rather than a delete-then-insert pass) means a partial seed run never leaves the map empty.

**Why pulled forward:** The plan's instinct ("F2 first because it's a bug") was right that F2 looks like a behaviour problem, but the underlying issue was data, not behaviour. Hand-reviewed `src/components/map-view.tsx`: the `useMemo`s on `plotted`, `positioned`, `filteredCompanies` all have the right deps and re-fire on state changes. With 150 scores now live, F2 may already be implicitly resolved — next chunk will verify on the live URL and either tick it off or fix what's actually broken.
