## Step 4 — scoring engine (server)
- `src/lib/scoring/index.ts` — `rescoreCompanyFrame(companyId, frameId, opts)`:
  - Pulls company + frame definition + recent publications + lobbying records.
  - Hashes the evidence set; short-circuits when unchanged unless `force=true`.
  - Calls `claude-3-5-sonnet-latest` with a strict-JSON system prompt that returns
    `{score, rationale, confidence, citation_ids}`.
  - Persists frame_scores (upsert) and replaces frame_score_evidence rows.
  - Falls back to centre-of-scale 3.0 with low confidence when API key is missing
    or the model call/parse fails (engine doesn't 500 the UI).
- `src/lib/scoring/index.ts` also exports `aggregateScore` (weighted mean,
  L=1/M=2/H=3) used by both server and client.
- `src/lib/scoring/useLiveAggregates.ts` — client hook for instant re-ranking
  when weights change, no server round trip.
- `src/app/actions-rescore.ts` — `rescoreCompanyAction` server action.
- `src/app/api/cron/rescore/route.ts` — nightly catch-up worker. Auth mirrors
  the other crons (x-vercel-cron / CRON_SECRET). Knobs: `?slugs=`, `?max=`,
  `?force=1`, `?dryRun=1`. Default budget 20 cells/run.
- `vercel.json` — added rescore cron at 08:00 UTC.

## Step 8 — Surprise modal (2026-06-23 13:55 UTC)

Three variants live: **Adjacency**, **Recency**, **Underrated**. Each runs a
simple rule against the snapshot (companies + scores + weights + company_notes
+ publications + lobby records) and then asks claude-haiku for a single
sentence "why" line. Without `ANTHROPIC_API_KEY` set, the engine returns a
deterministic template so the modal still works locally.

- **Adjacency:** anchor = highest-scoring engaged company on the user's
  top-weight frame (falls back to the absolute top scorer); candidates =
  others within 1.0 of the anchor's score on that frame, preferring
  unread (== no `company_notes` row).
- **Recency:** any company with a publication in the last 21d AND score
  ≥3.5 on the top-weight frame; freshest wins, light randomness across
  the top 3.
- **Underrated:** unread + ungrouped scorers ranked by lobbying-register
  footprint first, then frame-agnostic aggregate.

`<SurpriseButton />` lives in the nav (`site-shell`). Modal: 3 picks max
per open, round-robin tries unseen variants first, then loops. Anchor
metadata (`frame · score / daysAgo / aggregate`) is rendered so the cat's
reasoning stays auditable.
